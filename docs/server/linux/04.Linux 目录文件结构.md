# Linux 目录文件结构

以下是Linux系统根目录下的目录结构



```typescript
/
├─www //
├─var //全称是：vairiable，用于存储变动的文件，如日志文件，缓存等。特别重要，不能删除
├─usr //用户级，全称是:UNIX software resource ，主要存放软件程序和这些程序所需要的库。特别重要，不能删除
├─tmp //存储临时文件，长时间会自动删除无用文件
├─sys //系统功能目录，特别重要，不能删除
├─srv //用于存放软件源代码的
├─sbin //超级用户才能执行的指令文件
├─run // 程序运行时产生的文件，多数也是临时文件，但是这里的文件不会被系统自动删除
├─root //
├─proc //全称：process，进程，特别重要，不能删除
├─patch
├─opt //部分软件安装存储目录
├─mnt //全称：mount,挂载，一般用于挂载U盘等
├─media //多媒体文件目录
├─lib64 //全称：library 32 位库，一般是.so结尾的的库文件，特别重要
├─lib //全称：library 32 位库，一般是.so结尾的的库文件，特别重要
├─home// 所有普通用户的根目录在这个目录下，每个用户都用自己的桌面等目录，相当于windows的User目录
├─etc //存放系统配置文件，特别重要
├─dev //全称:devices，硬件设备控制文件，特别重要，不能动
├─boot// 开机启动需要的文件，不能动
├─bin //存放二进制的可执行文件，也就是命名。每个命令基本都是一个可执行代码文件，不能删除
```
